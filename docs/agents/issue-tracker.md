# Issue Tracker: GitHub

Issues and PRDs for this repo live as GitHub Issues in `geirsagberg/kartoffel`. Use the `gh` CLI for issue operations.

This repository is public. Do not include secrets, API keys, signing keys, private location data, or local configuration values in issues.

## Conventions

- Create issues with `gh issue create`.
- Read issues with `gh issue view <number> --comments`.
- List issues with `gh issue list`.
- Comment with `gh issue comment <number>`.
- Apply or remove labels with `gh issue edit`.
- Close issues with `gh issue close`.

When a skill says "publish to the issue tracker", create a GitHub Issue.
